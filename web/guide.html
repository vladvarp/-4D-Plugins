Структура файла

Каждый .md файл плагина состоит из двух частей: frontmatter (мета-данные в начале файла) и тела документа (Markdown-контент).

Файл должен находиться в папке plugins/ и иметь расширение .md. Для открытия страницы плагина используется URL вида:

URL страницы плагина 
plugin.html?md=plugins/название-плагина.md

Frontmatter

Frontmatter — это блок с мета-данными в самом начале файла. Он обрамляется тремя дефисами --- сверху и снизу. Используется формат ключ: значение.

Формат frontmatter 
---
ключ: значение
другой_ключ: "значение со спецсимволами"
список: [первый, второй, третий]
---

Здесь начинается основной контент...

Обязательные поля

Эти поля обязательно должны быть заполнены — они используются в hero-секции страницы.

Поле Тип Описание Пример
title строка Название плагина ScatterPro
tagline строка Краткое описание (1–2 предложения) Мощный инструмент для скаттеринга
version строка Текущая версия v2.4.1
author строка Имя автора или команды Иван Петров
updated строка Дата последнего обновления 15 мая 2025
Пример обязательных полей 
---
title: ScatterPro
tagline: Мощный инструмент для распределения объектов по поверхности с полным контролем над плотностью и масштабом.
version: v2.4.1
author: Иван Петров
updated: 15 мая 2025
---

Блоки Callout

Специальные блоки для выделения важной информации. Используется синтаксис :::[тип] … :::.

Доступные типы: info, warning, danger, tip.

Синтаксис callout блоков 
:::info
Информационное сообщение. Используй для подсказок и полезных ссылок.
:::

:::warning
Предупреждение. Используй для важных заметок, на которые нужно обратить внимание.
:::

:::danger
Критическое предупреждение. Используй для описания деструктивных действий.
:::

:::tip
Совет. Используй для лайфхаков и рекомендаций.
:::
ℹ️

Так выглядит info-блок на странице плагина.

Раскрывающиеся блоки

Два типа сворачиваемых секций: развёрнутый по умолчанию и свёрнутый. Первая строка — заголовок, остальное — содержимое (поддерживается базовый Markdown).

Развёрнутый по умолчанию

Синтаксис ::::details-open 
::::details-open
Расширенные настройки
Здесь можно описать продвинутые параметры плагина.
- Пункт первый
- Пункт второй
::::
Расширенные настройки

Так выглядит развёрнутый блок. Пользователь может свернуть его кликом по заголовку.

Свёрнутый по умолчанию

Синтаксис ::::details 
::::details
Часто задаваемые вопросы
**Работает ли плагин с Redshift?**
Да, полностью совместим с Redshift и Arnold.

**Нужен ли интернет?**
Нет, плагин работает офлайн.
::::
Часто задаваемые вопросы

Так выглядит свёрнутый блок — по умолчанию содержимое скрыто.

2. Скопируй плагин в `~/Library/Preferences/MAXON/Cinema 4D/plugins`

Пошаговые инструкции

Визуальная нумерация шагов для установки, настройки и т.д. Каждый шаг — ### с названием, следующие строки — описание.

Синтаксис ::::steps 
::::steps
### Скачай плагин
Нажми кнопку «Скачать» вверху страницы или используй [[button:Скачать|https://example.com/download]].

### Установи в Cinema 4D
Распакуй архив в папку `plugins` и перезапусти программу.

### Открой плагин
Перейди в меню **Extensions → My Plugin**.

    Распакуй архив в папку plugins и перезапусти программу.

  • Открой плагин

    Перейди в меню Extensions → My Plugin.

Видео

Встраивание видео с YouTube или Vimeo. Первая строка — URL, вторая (необязательно) — подпись под видео.

Синтаксис ::::video 
::::video
https://www.youtube.com/watch?v=dQw4w9WgXcQ
Демонстрация работы плагина
::::

Поддерживаются форматы URL: youtube.com/watch?v=, youtu.be/, vimeo.com/.

Фотоальбом (кнопка)

Кнопка, при нажатии на которую открывается полноэкранное модальное окно с фотоальбомом. Поддерживает пролистывание клавишами ← → и свайпами на мобильных устройствах.

Параметры

Параметр Формат Описание
-n -n'Название' Текст кнопки (по умолчанию «Галерея»)
-p -p'путь1','путь2',... Список путей к фото в одинарных кавычках через запятую
Синтаксис фотоальбома 
[[photo: -n'Галерея' -p'photo/photo1.png','photo/photo2.png','photo/photo3.png','photo/photo4.png']]
ℹ️

Кнопка работает внутри текста, таблиц и любых других блоков. В модальном окне используются клавиши ← → для навигации и Esc для закрытия.

Фотокарусель

Панель с каруселью фотографий внутри. Каждая фотография отображается поочерёдно с возможностью навигации кнопками, точками или свайпами.

Параметры

Параметр Формат Описание
-n -n[Заголовок] Заголовок панели (необязательный). Если не задан — панель без заголовка
-s -sШирина*Высота Размер панели в пикселях (например, -s300*500). Панель адаптируется к ширине контейнера
-c -c1 / -c2 / -c3 Выравнивание: -c1 по левому краю (по умолчанию), -c2 по центру, -c3 по правому краю
-asc -ascМИЛЛИСЕКУНДЫ Пауза между анимациями (например, -asc2000). Значение -asc0 — непрерывная прокрутка без пауз. Останавливается при наведении
-asp -aspМИЛЛИСЕКУНДЫ Скорость анимации перехода (например, -asp500). По умолчанию 500
Синтаксис фотокарусели 
===photo -n[Галерея] -s300*400
[[p:'photo/photo1.png']]
[[p:'photo/photo2.png']]
[[p:'photo/photo3.png']]
[[p:'photo/photo4.png']]
===

===photo -n[Центр] -s400*300 -c2
[[p:'photo/img1.png']]
[[p:'photo/img2.png']]
===

===photo -n[Автопрокрутка] -s400*300 -asc3000 -asp600
[[p:'photo/a.png']]
[[p:'photo/b.png']]
[[p:'photo/c.png']]
===
ℹ️

Изображения автоматически заполняют область карусели (object-fit: cover), поэтому карусель выглядит хорошо при любых соотношениях сторон. Карусель использует 3D-эффект — соседние фотографии видны на переднем плане с уменьшением и прозрачностью. Нажатие на любую фотографию открывает полноэкранный просмотр именно с неё. На мобильных устройствах поддерживается свайп-навигация.

Сравнение фото

Блок для интерактивного сравнения 2–3 фотографий с перетаскиваемыми разделителями. Идеально подходит для показа «до/после», этапов обработки и сравнения вариантов.

Параметры

Параметр Формат Описание
-n -n[Заголовок] Заголовок блока (по умолчанию «Сравнение»)
-s -sШирина*Высота Размер блока в пикселях (например, -s600*400)
-c -c1 / -c2 / -c3 Выравнивание: -c1 по левому краю (по умолчанию), -c2 по центру, -c3 по правому краю
-asc -ascМИЛЛИСЕКУНДЫ Пауза между анимациями (например, -asc2000). Значение -asc0 — непрерывная прокрутка без пауз. Останавливается при наведении
-asp -aspМИЛЛИСЕКУНДЫ Скорость анимации перехода (например, -asp500). По умолчанию 500
-d -d[Подпись] Подпись под фотографией (отображается поверх изображения)

Сравнение двух фото

Синтаксис: 2 фото 
===leveling -n[До и после] -s600*400
[[lv:'photo/before.png']] -d[До]
[[lv:'photo/after.png']] -d[После]
===

===leveling -n[Центрированное] -s600*400 -c2
[[lv:'photo/before.png']] -d[До]
[[lv:'photo/after.png']] -d[После]
===

===leveling -n[Авто] -s600*400 -asc2500 -asp800
[[lv:'photo/before.png']] -d[До]
[[lv:'photo/after.png']] -d[После]
===

Сравнение трёх фото

Синтаксис: 3 фото 
===leveling -n[Этапы обработки] -s600*400
[[lv:'photo/step1.png']] -d[Без обработки]
[[lv:'photo/step2.png']] -d[Средняя]
[[lv:'photo/step3.png']] -d[Финальная]
===
ℹ️

Поддерживается от 2 до 3 фотографий. Разделители перетаскиваются мышью (или пальцем на мобильных устройствах). Подписи отображаются поверх изображений и автоматически позиционируются по центру каждой секции.

Модальные окна

Модальные окна позволяют скрыть большой контент за кнопкой и показывать его во всплывающем окне по клику. Содержимое поддерживает весь Markdown-синтаксис, включая заголовки, списки, таблицы, код, а также шапку с иконкой, названием и описанием.

Синтаксис

Модальное окно состоит из двух частей:

  1. Кнопка — [[mdb:'Название кнопки'-n'ID'-sШ]]
  2. Содержимое — блок <mdc-n'ID'-sШ*В> ... </mdc>

Параметры кнопки

Параметр Формат Описание
n -n'ID' Уникальный идентификатор модального окна (связывает кнопку и содержимое)
s -sШирина Ширина кнопки в пикселях (необязательный). Например, -s90
b -b1 – -b9 Стиль кнопки: -b1 акцент, -b2 контур, -b3 красный, -b4 зелёный, -b5 призрачный, -b6 градиент, -b7 тёмный, -b8 крупный, -b9 плавный

Параметры содержимого

Параметр Формат Описание
n -n'ID' Идентификатор (должен совпадать с кнопкой)
s -sШирина*Высота Размер окна в пикселях (например, -s400*500). Высота — максимальная, окно может быть меньше если контент короче

Шапка модального окна

В начале содержимого можно указать необязательные поля icon, title, tagline — они отобразятся в шапке окна. Эти поля парсятся из первых строк блока и не попадают в тело контента.

Пример модального окна 
[[mdb:'Открыть описание'-n'window_1'-s120]]

Содержимое может быть где угодно в документе:

<mdc-n'window_1'-s400*500>
icon: ico/ObjectRenamerPRO.png
title: Object Renamer PRO
tagline: Плавающая немодальная панель для пакетного переименования.

| Параметр | Тип    | Описание          |

| suffix   | string | Суффикс имени     |

</mdc>
ℹ️

Блок <mdc> может находиться в любом месте документа — он не отображается на странице, а только хранит содержимое. Поле tagline отображается как тултип при наведении на кнопку ? в шапке окна. Модальное окно закрывается клавишей Esc или кликом по фону.

Изображения и значки

Инлайн-изображения и иконки вставляются прямо в текст или в любой блок с помощью синтаксиса [[ico:'путь'|ШxВ]]. Путь указывается в одинарных кавычках, размер — через x или *.

Синтаксис

Форматы [[ico:...]] 

[[ico:'plugins/photo.png'-200]]      — высота 200px, ширина пропорционально
[[ico:'plugins/photo.png'-x]]        — автоподгонка
[[ico:'plugins/icon.png']]           — по умолчанию 36x36
[[ico:'plugins/photo.png'-300-c2]]   — по центру
Формат Описание
[[ico:'path'-H]] Только высота H — ширина вычисляется пропорционально
[[ico:'path'-x]] Автоподгонка
[[ico:'path']] Размер по умолчанию — 36×36 px
-c1 / -c2 / -c3 Выравнивание: -c1 по левому (по умолчанию), -c2 по центру, -c3 по правому. Добавляется после размера
ℹ️
Поле icon во frontmatter тоже поддерживает PNG/SVG/JPG: достаточно указать путь к файлу (например, icon: plugins/my-icon.png) — он автоматически отрисуется как изображение в карточке плагина и в шапке страницы плагина. Если значение не похоже на путь к файлу, оно отображается как emoji/символ, как и раньше.

Пустые ячейки

Если в ячейке стоит только символ -, она отображается как пустая. Удобно для ячеек без значения:

Синтаксис пустых ячеек 

| Параметр | Тип    | По умолчанию |
|----------|--------|--------------|
| density  | float  | 1.0          |
| name     | string | -            |
| hidden   | -      | -            |

Ячейки с - будут отображены пустыми — без текста и без дефиса.

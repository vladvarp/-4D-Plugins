## О плагине
===photo -s400*500
[[p:'image/SelectionSet/SelectionSet.png']]
===
Selection Sets — инструмент для сохранения и восстановления наборов выделения объектов. В сцене с десятками или сотнями объектов постоянное переключение между разными группами выделения отнимает время. Плагин позволяет сохранить текущее выделение как именованный набор и восстановить его в один клик.

Каждый набор хранится как тег на нулевом объекте внутри контейнера `Selection Sets` в корне сцены. Ссылки на объекты сохраняются напрямую, поэтому наборы переживают переименование и корректно работают при передаче файла коллеге.

## Возможности

::::features
### 📋 Сохранение набора
Выдели нужные объекты, нажми **Новый набор** — появится диалог ввода имени (по умолчанию «Пользовательский набор»). Нулевой объект с тегом автоматически помещается в контейнер **Selection Sets**.

### ⚡ Восстановление выделения
Нажми **Выделить** — текущее выделение сбрасывается, а объекты из выбранного набора выделяются. Никакого сложения с предыдущим выделением.

### ➕ Добавление и удаление объектов
Кнопки **+ Добавить** и **− Убрать** позволяют изменять состав набора без его пересоздания. Выдели объекты в сцене и нажми нужную кнопку.

### 🗑️ Удаление набора
Кнопка **Удалить набор** удаляет тег и нулевой объект целиком. Если тег был подвешен вручную на пользовательском объекте — удаляется только тег, объект остаётся.

### 🏷️ Переименование
Кнопка **Переименовать** меняет имя набора. Для автосозданных нулевых объектов переименовывается и сам объект. Для пользовательских объектов — только поле UserData тега.

### 🧹 Очистка
Кнопка **Очистить** убирает все ссылки на объекты из набора, но оставляет сам набор. Полезно для пересборки состава.

### 🖱️ Двойной клик по тегу
Двойной клик по тегу Selection Set Tag в Object Manager открывает окно плагина — быстрый доступ к управлению набором.
::::

## Использование

### Создание набора

1. Выдели объекты в Object Manager или вьюпорте
2. Открой плагин через `Extensions` → `Selection Sets`
3. Нажми **Новый набор**
4. Введи имя (или оставь «Пользовательский набор»)
5. Набор появится в выпадающем списке

### Восстановление выделения

1. Открой плагин
2. Выбери нужный набор из выпадающего списка
3. Нажми **Выделить** — текущее выделение заменится на сохранённое

### Добавление объектов в набор

1. Выдели объекты, которые нужно добавить
2. Выбери набор в списке
3. Нажми **+ Добавить**

:::tip
Используй наборы для группировки объектов по функциональным группам: камеры, освещение, декор, персонажи. Это ускоряет переключение между рабочими контекстами.
:::

## Интерфейс диалога

::::details-open
Элементы управления

| Элемент | Описание |
|---------|----------|
| **Набор** | Выпадающий список всех наборов в сцене с указанием количества объектов |
| **Выделить** | Сбрасывает текущее выделение и выделяет объекты из выбранного набора |
| **+ Добавить** | Добавляет выделенные объекты в выбранный набор |
| **− Убрать** | Убирает выделенные объекты из набора |
| **Очистить** | Удаляет все ссылки на объекты из набора (сам набор остаётся) |
| **Удалить набор** | Удаляет набор и нулевой объект (для автосозданных) или только тег (для ручных) |
| **Новый набор** | Создаёт набор из текущего выделения |
| **Переименовать** | Меняет имя набора |
| **Объекты в наборе** | Список объектов в текущем наборе с указанием типа |

## Хранение данных

::::details
Структура в Object Manager

```
Selection Sets          ← нулевой объект с иконкой плагина
├── My Set              ← нулевой объект с иконкой тега
│   └── [Selection Set Tag]
├── Another Set
│   └── [Selection Set Tag]
└── ...
```

Каждый набор — это тег **Selection Set Tag** (`PLUGIN_ID = 1068966`) на нулевом объекте. Тег хранит в UserData:

| Поле | Тип | Описание |
|------|-----|----------|
| `Set Name` | string | Имя набора |
| `Obj_1`, `Obj_2`, … | BaseListLink | Ссылки на объекты (до 256) |

Ссылки хранятся как прямые указатели на объекты — набор корректно работает при переименовании объектов.
::::

## Особенности поведения

- **Авто-созданные наборы** (через «Новый набор») — нулевые объекты с иконкой тега внутри контейнера **Selection Sets** с иконкой плагина. При удалении удаляется и нулевой объект.
- **Ручные наборы** (тег на пользовательском объекте) — при удалении удаляется только тег. Объект не удаляется и не переименовывается.
- **Переименование** — для авто-созданных нулевых объектов переименовывается и сам объект. Для пользовательских объектов — только UserData тега.

:::warning
Максимальное количество объектов в одном наборе — 256. Это ограничение связано с хранением ссылок в UserData тега.
:::

::::details
Часто задаваемые вопросы

**Объект пропал из набора после переименования.**
Наборы хранят прямые ссылки на объекты, а не их имена. Переименование не влияет на набор. Если объект был удалён из сцены, ссылка станет пустой и объект исчезнет из списка.

---
**Как отличить автосозданный набор от ручного?**
Автосозданные наборы — это нулевые объекты внутри контейнера **Selection Sets** с иконкой тега. Ручные наборы — это теги на любых объектах сцены без привязки к контейнеру.

---
**Можно ли перемещать наборы между документами?**
Да, скопируй нулевой объект с тегом из одного документа в другой. Ссылки на объекты будут валидны только если объекты тоже скопированы.
::::

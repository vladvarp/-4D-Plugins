Action Recorder — инструмент для записи и воспроизведения операций Cinema 4D в виде Python-скриптов. Плагин перехватывает все изменения сцены (добавление/удаление объектов, трансформации, изменения параметров) и преобразует их в исполняемый код.

Записанные операции сохраняются как `.py` файлы в папке `~/Documents/ActionRecorder/` и могут быть запущены повторно в любом проекте. Это позволяет автоматизировать рутинные задачи: создание стандартных объектов, позиционирование, настройку рендера.

:::info
Плагин находится во вкладке AR в главном меню. Состоит из трёх команд: Лог событий, Менеджер операций и Воспроизвести последний.
:::

## Возможности

::::features
### 🔴 Запись изменений сцены
Автоматический перехват всех действий: добавление/удаление объектов, трансформации, модификация параметров. Запись ведётся через глобальный хук `EVMSG_CHANGE`.

### 📋 Лог событий
Просмотр всех зафиксированных изменений в виде форматированного Python-кода с временны́ми метками. Лог обновляется в реальном времени.

### 📂 Коллекции и операции
Организация операций в именованные коллекции (папки). Каждая операция — отдельный `.py` файл с кодом.

### ▶️ Воспроизведение
Запуск любой операции одной кнопкой. Код выполняется в контексте активного документа с доступом к `c4d` и `doc`.

### ✎ Редактирование кода
Встроенный редактор Python-кода с подсветкой синтаксиса. Можно вручную отредактировать записанный код перед запуском.

### ⚡ Быстрое воспроизведение
Кнопка «Воспроизвести последний» запускает последнюю использованную операцию без открытия менеджера.
::::

## Использование

### Запись операции

1. Открой Менеджер операций: вкладка `AR` → `Менеджер операций`
2. Создай новую коллекцию или выбери существующую
3. Нажми **+ Новый** для создания операции
4. Нажми **●** рядом с операцией для начала записи
5. Выполняй любые действия в сцене — все изменения записываются автоматически
6. Нажми **■** для остановки записи

:::tip
Запись работает через глобальный хук `EVMSG_CHANGE`. Добавление примитивов через палитру автоматически распознаётся как `c4d.CallCommand()`.
:::

### Воспроизведение

1. Выбери операцию в списке
2. Нажми **▶** для запуска
3. Код выполнится в контексте текущего документа

Или используй кнопку **AR: Воспроизвести последний** для быстрого запуска последней операции.

### Редактирование кода

1. Выбери операцию в списке
2. Отредактируй код в правой панели
3. Нажми **Сохранить код** или используй горячую клавишу
4. Статус сохранения отображается в нижней части редактора

:::warning
При ручном редактировании кода убедись, что синтаксис Python корректен. Ошибки в коде приведут к сбою при воспроизведении.
:::

## Менеджер операций

::::details-open
Элементы управления

| Элемент | Описание |
|---------|----------|
| **Коллекция** | Выпадающий список коллекций (папок с операциями) |
| **+ Новая** | Создать новую коллекцию |
| **Удалить** | Удалить текущую коллекцию и все её операции |
| **+ Новый** | Создать новую операцию в текущей коллекции |
| **▶** | Воспроизвести выбранную операцию |
| **●/■** | Начать/остановить запись |
| **✎** | Переименовать операцию |
| **✕** | Удалить операцию |
| **Код операции** | Редактор Python-кода с подсветкой синтаксиса |
| **Сохранить код** | Сохранить изменения из редактора в файл |
| **Восстановить** | Откатить изменения из редактора к сохранённой версии |
::::

## Лог событий

::::details-open
Элементы управления

| Элемент | Описание |
|---------|----------|
| **Статус** | Индикация активности лога |
| **Очистить** | Очистить текущий лог (не удаляет файлы операций) |
| **Сохранить как операцию** | Сохранить содержимое лога как новую операцию в коллекции |
::::

:::info
Лог обновляется автоматически при каждом изменении сцены. Каждое событие помечено временны́й меткой в формате `HH:MM:SS`.
:::

## Структура хранения

::::details
Файловая система

```
~/Documents/ActionRecorder/
├── Коллекция/          ← папка коллекции
│   ├── Операция1.py    ← файл операции
│   └── Операция2.py
├── Другая коллекция/
│   └── ...
└── _last_action.json   ← мета-информация о последней операции
```

Каждый `.py` файл операции содержит:

```python
# Action: Имя операции
# Recorded: 2026-06-29 14:30:00
import c4d

c4d.CallCommand(5159)  # Куб
obj = doc.SearchObject('Куб')
if obj:
    obj.SetAbsPos(c4d.Vector(100.0, 0.0, 0.0))
    c4d.EventAdd()
```
::::

## Ограничения

- Запись работает только при открытом диалоге Лога или Менеджера
- Сложные операции (программное создание объектов через Python) могут не распознаваться корректно
- Операции не зависят от сцены — при воспроизведении объекты создаются в текущем документе
- Имена объектов в коде привязаны к именам при записи — переименование объектов в сцене может слить поиск

:::warning
При воспроизведении операции создают новые объекты параллельно существующим. Если в сцене уже есть объект с таким именем — `SearchObject` вернёт первый найденный, что может привести к неожиданным результатам.
:::

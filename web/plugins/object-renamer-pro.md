## О плагине

Object Renamer PRO — инструмент для наведения порядка в сцене. Плагин решает одну из самых рутинных задач 3D-художника: быстрое и массовое переименование объектов с полным контролем над результатом.

В отличие от ручного переименования по одному объекту, плагин обрабатывает любое количество выделенных объектов одновременно. Все операции — найти и заменить, добавить префикс, пронумеровать, изменить регистр, транслитерировать — применяются пакетно и каждая создаёт **один шаг отмены**. Это значит, что `Ctrl+Z` всегда откатит всё пакетное действие целиком, а не по одному объекту.

Панель открывается как немодальное плавающее окно и не блокирует работу в сцене.

## Возможности

::::features
### 🔍 Умное выделение
Выделяй объекты по имени (точное совпадение или фрагмент), по типу, по тегу или по материалу. Область поиска настраивается: глобально по всей сцене, локально на одном уровне иерархии или рекурсивно.

### ✏️ Найти и заменить
Замена любой подстроки в именах выделенных объектов. Поддерживает чувствительность к регистру, работает пакетно по всем выделенным объектам за одно нажатие.

### 🏷️ Префиксы и суффиксы
Добавление и удаление произвольного префикса или суффикса. Автоматический суффикс или префикс типа объекта: `_MESH`, `_CAM`, `_LIGHT`, `_NULL` и другие — более 60 типов.

### 🔢 Автонумерация
Добавляет порядковый номер в конец или начало имени. Настраиваемый разделитель, стартовый номер и количество знаков (`01`, `001` и т.д.). Удаление числовых суффиксов и префиксов одним кликом.

### 🔠 Регистр и очистка
Перевод в ВЕРХНИЙ, нижний, Заглавный или первый-заглавный регистр. Замена пробелов на подчёркивания и обратно. Удаление всех пробелов. Транслитерация RU → EN по стандартной таблице.

### 🔗 Синхронизация с материалами
Переименование объекта по имени первого назначенного материала (и наоборот — переименование материала по имени объекта). Удобно для поддержания консистентного нейминга.

### ⇅ Сортировка объектов
Сортировка объектов внутри уровня иерархии по имени, типу, числу дочерних объектов или количеству тегов. Каждая кнопка сортировки переключается между прямым и обратным порядком. Отдельная сортировка тегов на выделенных объектах.

### 👁️ Живой предпросмотр
Скрытая панель предпросмотра (включается кнопкой ⊙) обновляется каждые 400 мс и показывает, как будут выглядеть имена объектов после применения текущей операции — до нажатия кнопки.
::::

## Установка

1. Скачай программу `4DPluginInstaller.exe` по кнопке выше
2. Выбери папку плагинов Cinema 4D
3. Установи пакет **Object Renamer PRO**
4. Перезапусти Cinema 4D
5. Плагин появится в меню **Extensions → Object Renamer PRO**

:::info
Папка плагинов Cinema 4D на Windows: `C:\Users\<имя>\AppData\Roaming\Maxon\Cinema 4D R26\plugins`
:::

## Структура интерфейса

Панель разделена на вкладки. Каждая вкладка отвечает за отдельную группу операций:

| Вкладка | Содержимое |
|---|---|
| **Выделение** | Поиск по имени, выделение по типу, тегу, материалу, настройка области |
| **Сортировка** | Сортировка объектов и тегов на уровне иерархии |
| **Имена** | Найти и заменить, добавление и удаление префикса/суффикса |
| **Нумерация** | Автонумерация в начало или конец имени, удаление чисел |
| **Регистр** | Смена регистра, замена символов, транслитерация, синхронизация с материалами |

В нижней части панели — строка состояния с результатом последней операции и кнопка ⊙ для показа/скрытия предпросмотра.

## Область действия (Scope)

Большинство операций выделения работают относительно **первого выделенного объекта** и задаются выпадающим списком **Область**:

| Значение | Описание |
|---|---|
| **Глобально** | Все объекты сцены, включая вложенные |
| **Локально** | Только объекты на том же уровне иерархии (дочерние того же родителя) |
| **Рекурсивно** | Текущий уровень иерархии и все его дочерние объекты |

:::tip
Для большинства задач удобно использовать режим **Локально** — он не затрагивает объекты из других иерархий и даёт предсказуемый результат.
:::

## Операции по вкладкам

### Выделение

- **Поиск по имени** — введи строку в поле поиска и нажми **Точное совпадение** или **Фрагмент**. Чекбокс **Учитывать регистр** управляет чувствительностью.
- **Выделить по совпадению с первым объектом** — выдели один объект-эталон, затем нажми нужную кнопку:
- **По имени** — выделяет все объекты с таким же именем в заданной области
- **По типу** — выделяет все объекты того же типа (камеры, меши, нулли и т.д.)
- **По первому тегу** — выделяет объекты, у которых есть тег того же типа, что и первый тег эталона
- **По первому материалу** — выделяет объекты с тем же материалом; если у эталона выделен конкретный тег-текстуры — берётся материал именно из него

### Имена (найти и заменить, префиксы/суффиксы)

- **Применить замену** — заменяет подстроку из поля **Найти** на строку из поля **Заменить** у всех выделенных объектов
- **+ Префикс / + Суффикс** — добавляет введённый текст в начало или конец имени
- **- Префикс / - Суффикс** — удаляет введённый текст, если имя начинается или заканчивается на него
- **+ Преф. типа** — добавляет префикс по типу объекта (`CUBE_`, `CAM_`, `NULL_` и т.д.)
- **+ Суфф. типа** — добавляет суффикс по типу объекта (`_CUBE`, `_CAM`, `_NULL` и т.д.)

### Нумерация

Настройки нумерации:

| Параметр | Описание |
|---|---|
| **Разд-ль** | Символ между именем и числом (по умолчанию `_`) |
| **С:** | Стартовый номер (по умолчанию `1`) |
| **Знаков:** | Количество знаков в числе: `2` → `01`, `3` → `001` |

Кнопки:

- **+ Число в начало** — `01_Имя`, `02_Имя`, ...
- **+ Число в конец** — `Имя_01`, `Имя_02`, ...
- **- Числ. префикс** — удаляет числовой префикс вида `01_`, `001-` и т.п.
- **- Числ. суффикс** — удаляет числовой суффикс вида `_01`, `.001` и т.п.

### Регистр

- **ВЕРХНИЙ / нижний / Заглавные** — стандартные варианты регистра
- **Первый заглавный** — только первая буква становится заглавной, остальные не меняются (Не сработает если имя `ВЕРХНИЙ` или `Заглавные`, сначала нужно нажать `нижний`)
- **Пробелы → _ / _ → Пробелы** — замена символов
- **Убрать пробелы** — удаляет все пробелы из имени
- **RU → EN (транслит)** — транслитерирует кириллицу в латиницу по стандартной таблице (ж→zh, щ→sch, ю→yu и т.д.)
- **Объект → Материал** — переименовывает материал объекта по имени самого объекта
- **Объект ← Материал** — переименовывает объект по имени его первого материала

### Сортировка

Сортировка работает внутри **уровня иерархии** первого выделенного объекта. Каждая кнопка сортировки является toggle: первое нажатие — прямой порядок, второе — обратный.

| Кнопка | Описание |
|---|---|
| **Имя А→Я / Я→А** | По алфавиту |
| **Тип А→Я / Я→А** | По строковому коду типа объекта |
| **Дочерних ↓/↑** | По количеству прямых дочерних объектов |
| **Число тегов ↓/↑** | По количеству тегов на объекте |
| **⇅ Обратить порядок** | Разворачивает текущий порядок объектов |
| **Имя А→Я (теги)** | Сортировка тегов на выделенных объектах по имени |
| **Тип А→Я (теги)** | Сортировка тегов по числовому ID типа, затем по имени |

:::warning
Сортировка меняет физический порядок объектов в иерархии сцены. Это действие поддерживает отмену через `Ctrl+Z`.
:::

## Предпросмотр

Нажми кнопку ⊙ в правом нижнем углу панели, чтобы показать или скрыть панель предпросмотра. Предпросмотр обновляется автоматически каждые 400 мс и отображает:

- На вкладке **Выделение** — текущий пул объектов и область поиска
- На вкладке **Имена** — как изменятся имена при применении текущей замены
- На вкладке **Нумерация** — как будет выглядеть имя с числом в начале и в конце
- На вкладке **Регистр** — ВЕРХНИЙ, нижний и Заглавный варианты имён объектов
- На вкладке **Сортировка** — текущие объекты на уровне с их типом и числом тегов

:::tip
Предпросмотр показывает до 6 объектов. Если выделено больше — в конце списка появляется строка «… ещё N объектов».
:::

## Механика отмены

Каждая операция переименования или выделения оборачивается в блок `StartUndo() / EndUndo()`. Это означает, что `Ctrl+Z` всегда откатывает **всё пакетное действие** за один шаг, а не объект за объектом.

Операции сортировки также создают единый шаг отмены — вся перестановка объектов откатывается целиком.

## Суффиксы и префиксы типов

Плагин поддерживает более 60 типов объектов Cinema 4D. Примеры:

| Тип объекта | Код |
|---|---|
| Null | `NULL` |
| Полигональный меш | `MESH` |
| Камера | `CAM` |
| Источник света | `LIGHT` |
| Сплайн | `SPLN` |
| Extrude | `EXT` |
| Subdivision Surface | `SDS` |
| Symmetry | `SYM` |
| Joint | `JNT` |
| Volume Builder | `VOLB` |

Для неизвестных типов используется код `OBJ`.

## О плагине

Snapshot — однокнопочный инструмент для «запекания» анимации в геометрию. Выдели анимированный объект, запусти плагин, введи нужное количество кадров — и Cinema 4D пройдёт по всей временной шкале, на каждом кадре объединит объект в меш и соберёт все копии под один нулевой объект **Снепшот**. В финале весь набор сразу объединяется в единый полигональный объект и удаляется временная папка.

Плагин незаменим при работе с процедурными деформациями, клонерами, симуляцией ткани или любой другой анимацией, которую нужно зафиксировать как статичную геометрию для экспорта или дальнейшего редактирования.

:::info
Плагин активен только при наличии выделенного объекта в сцене.
:::

## Возможности

::::features
### 📷 Снепшот по кадрам
Проходит по каждому кадру анимации и создаёт объединённый полигональный меш — фиксирует состояние объекта со всеми деформациями, симуляцией и процедурными эффектами.

### 🔢 Масштабирование временной шкалы
Перед созданием снепшотов плагин запрашивает новое количество кадров и пропорционально масштабирует все ключевые кадры анимации — треки объектов, тегов и дочерних элементов обрабатываются рекурсивно.

### 📦 Автоматическая сборка
Все промежуточные мешы (`Снепшот_0`, `Снепшот_1`, …) помещаются в нулевой объект **Снепшот**, после чего немедленно объединяются командой **Объединить объекты и удалить** — результат готов к экспорту, Ctrl+Z возвращает на шаг назад, где обекты находятся внутри нулевого.

### ⚡ Автоматическое обновление сцены
На каждом кадре вызывается `ExecutePasses` с полными флагами пересчёта, что гарантирует корректный снимок даже для сложных процедурных стеков с симуляцией.
::::

## Использование

### Основной сценарий

1. `Выдели объект` в сцене — тот, анимацию которого нужно запечь в геометрию
2. Запусти `Extensions` → `Snapshot`
3. В диалоге введи `новое количество кадров` (например, `120`)
4. Дождись завершения — в консоли будет отображаться лог вида `Объект создан на кадре N`
5. В иерархии появится единый полигональный объект с полной геометрией анимации

:::tip
Запускай плагин с небольшим числом кадров (10–30) для быстрой проверки результата перед финальным запуском на полной длине анимации.
:::

::::details
Что происходит внутри

| Шаг | Действие |
|-----|----------|
| 1 | Диалог запрашивает новое количество кадров |
| 2 | Вычисляется `scale_factor = новые_кадры / старые_кадры` |
| 3 | Все ключи анимации (объекты, теги, дочерние) масштабируются рекурсивно |
| 4 | Временная шкала проекта устанавливается на новое значение |
| 5 | Плейхед переходит на кадр 0 |
| 6 | Создаётся нулевой объект `Снепшот` |
| 7 | Цикл по кадрам: обновление сцены → объединение → переименование → вложение в папку |
| 8 | Плейхед возвращается на кадр 0, выделяется папка `Снепшот` |
| 9 | Команда `Объединить объекты и удалить` собирает финальный меш |
| 10 | Временная шкала возвращается к исходной длине |
::::

### Масштабирование анимации

Плагин умеет растягивать или сжимать анимацию перед созданием снепшотов. Это работает через пропорциональный пересчёт временных меток всех ключей:

```python
scale_factor = new_frame_count / old_frame_count
scaled_time = BaseTime(key.GetTime().GetFrame(fps) * scale_factor, fps)
```

Рекурсивная функция `scale_all_keys` обходит все объекты сцены, их теги и дочерние элементы — ни один трек не будет пропущен.

:::warning
Масштабирование изменяет временны́е метки ключей **для всей сцены**, а не только для выделенного объекта.
:::

## Ограничения

- Плагин обрабатывает **активный выделенный объект** (`GetActiveObject`).
- Введённое число кадров должно быть **целым положительным числом** — иначе отобразится ошибка валидации.

:::danger
Не запускай плагин на сценах с очень большим числом кадров (500+) без предварительного теста — операция необратима, а объединение сотен мешей может занять значительное время.
:::

## О плагине

Camera Resolution Manager — инструмент для тех, кто работает со сценами, содержащими несколько камер с разными форматами вывода. Вместо того чтобы каждый раз лезть в настройки рендера и вручную вбивать ширину и высоту, плагин позволяет заранее привязать нужное разрешение к каждой камере и переключаться между ними в один клик.

Данные хранятся прямо в User Data объекта камеры, поэтому разрешение не теряется при сохранении сцены и корректно работает в команде — любой, кто откроет файл с установленным плагином, увидит те же настройки.

## Возможности

::::features
### 🎥 Разрешение на камеру
Каждой камере сцены можно задать собственное разрешение. При активации камеры через плагин оно мгновенно применяется к настройкам рендера.

### 📐 Готовые пресеты
15 встроенных форматов: HD 720p, FHD 1080p, 2K, 4K UHD, 4K DCI, Instagram (1:1, 4:5, Stories), Twitter/X, Facebook, A4 (портрет и альбом), Square 1:1, Cinemascope. Любой нестандартный размер задаётся вручную как Custom.

### ⇄ Swap — мгновенный поворот кадра
Кнопка ⇄ меняет ширину и высоту местами — удобно переключаться между портретным и альбомным режимами без ручного ввода.

### ∝ Сохранение пропорций
Чекбокс ∝ блокирует соотношение сторон: при изменении ширины высота пересчитывается автоматически, и наоборот.

### ⟳ Живое обновление
Панель следит за сценой в реальном времени: добавление, удаление или переименование камер отражается без перезапуска плагина.

### 💾 Данные в сцене
Разрешение хранится в User Data камеры (поля `CamRes_W`, `CamRes_H`, `CamRes_Ratio`). Файл сцены самодостаточен — данные не теряются при переносе.
::::

## Установка

1. Скачай программу `4DPluginInstaller.exe` по кнопке выше
2. Выбери папку плагинов Cinema 4D
3. Установи плагин Camera Resolution Manager
4. Перезапусти Cinema 4D
5. Плагин появится в меню **Extensions → Resolution Manager**

:::info
Папка плагинов Cinema 4D на Windows: `C:\Users\<имя>\AppData\Roaming\Maxon\Cinema 4D R26\plugins`
:::

## Интерфейс панели

Панель открывается как немодальное плавающее окно и отображает список всех камер сцены. Каждая строка содержит:


| Элемент                 | Описание                                                                        |
| ----------------------- | ------------------------------------------------------------------------------- |
| `▶ Имя камеры`          | Маркер `▶` показывает камеру, активную в редакторе в данный момент             |
| Поле **W**              | Ширина кадра в пикселях (1–99999)                                               |
| Чекбокс **∝**           | Фиксация соотношения сторон                                                     |
| Кнопка **⇄**            | Обмен значений ширины и высоты                                                  |
| Поле **H**              | Высота кадра в пикселях (1–99999)                                               |
| Выпадающий список       | Быстрый выбор из 15 пресетов                                                    |
| Кнопка **Активировать** | Применяет разрешение камеры к рендеру и переключает вид редактора на эту камеру |

В нижней части панели постоянно отображается статус: имя активной камеры, количество камер в сцене и текущее разрешение рендера.

## Быстрый старт

1. Открой панель через **Extensions → Resolution Manager**
2. В строке нужной камеры введи значения **W** и **H** или выбери пресет из выпадающего списка
3. Нажми **Активировать** — камера станет активной в редакторе, а разрешение рендера обновится

:::tip
Если ты работаешь с несколькими форматами одновременно (например, квадрат для Instagram и 16:9 для видео), настрой разные камеры заранее и переключайся между ними кнопкой **Активировать** — не придётся каждый раз менять рендер-настройки вручную.
:::

## Механика хранения данных

Плагин сохраняет разрешение прямо в **User Data** объекта камеры. Для каждой камеры создаются три скрытых поля:


| Поле           | Тип     | Описание           |
| -------------- | ------- | ------------------ |
| `CamRes_W`     | integer | Ширина кадра       |
| `CamRes_H`     | integer | Высота кадра       |
| `CamRes_Ratio` | boolean | Фиксация пропорций |

Поля создаются автоматически при первом вводе данных. Если поля уже существуют — плагин просто читает из них значения. Это означает, что данные не теряются при сохранении, открытии на другом компьютере или передаче файла коллеге (при условии, что плагин установлен).

## Реакция на изменения в сцене

Панель автоматически перестраивается в следующих ситуациях:

- Добавлена или удалена камера
- Камера переименована
- Пользователь вручную изменил разрешение рендера в настройках (кнопка **Активировать** сбрасывается)
- Открыт другой проект

:::warning
Если разрешение рендера изменено вручную (не через плагин), кнопка активированной камеры сбросится в состояние **Активировать**. Это сделано намеренно: плагин не перезаписывает изменения, внесённые снаружи.
:::

## Часто задаваемые вопросы

### Почему кнопка «✓ Активна» сбросилась?

Это происходит если разрешение рендера было изменено вручную в настройках Cinema 4D или камера была переключена вручную. Плагин отслеживает совпадение сохранённых значений камеры с текущим разрешением рендера.

### Камеры не отображаются в панели

Убедись, что объекты являются именно объектами-камерами (`Ocamera`). Нулевые объекты, Compositing Camera или камеры внутри риг-иерархий могут отображаться, только если плагин их видит при обходе сцены.

### Данные камеры пропали после закрытия Cinema 4D

Убедись, что сцена была сохранена после ввода данных. User Data сохраняется вместе со сценой и не теряется между сессиями.
